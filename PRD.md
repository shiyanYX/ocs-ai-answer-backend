# OCS AI 搜题后端 - Web 配置管理平台

## 1. Concept & Vision

一个现代化、高效的Web配置管理平台，为OCS刷题脚本的AI搜题后端提供直观的配置界面。采用深邃的暗色调设计，融合科技感与专业性，让用户能够轻松管理AI配置、测试API接口、监控系统状态。界面简洁而不简单，每个细节都经过精心打磨。

## 2. Design Language

### 美学方向
采用 **"科技实验室"** 设计风格 - 深邃的暗色调配合渐变光效，营造专业而神秘的科技感。界面如同一个高端的控制系统面板，每个交互都带有细腻的反馈。

```css
--bg-primary: #0a0a0f;        /* 深邃黑 */
--bg-secondary: #12121a;      /* 次级背景 */

### 字体选择
- **主标题**: `'Space Grotesk', sans-serif` - 科技感强
- **正文**: `'Inter', -apple-system, sans-serif` - 清晰易读
- **代码/配置**: `'JetBrains Mono', monospace` - 等宽专业

### 空间系统
- 基础间距单位: 8px
- 卡片内边距: 24px
- 组件间距: 16px
- 圆角: 12px (卡片), 8px (按钮/输入框)

### 动效哲学
- **入场动画**: 渐显上移，duration 400ms，ease-out
- **交互反馈**: scale(0.98) + 微弱发光，150ms
- **状态切换**: 平滑渐变，200ms
- **加载状态**: 优雅的脉冲光效
- **成功反馈**: 短暂闪烁 + 图标变化

## 4. Features & Interactions

### 4.1 健康状态监控
- 实时显示后端服务状态（绿色脉冲=在线，红色=离线）
- AI服务启用状态指示
- 最后检查时间显示
- 自动刷新间隔：30秒

### 4.2 配置管理
**API配置区块**
- API Key 输入框（密码模式，可切换显示）
- API Base URL 输入框
- 模型选择下拉框（GPT-3.5-Turbo, GPT-4, GPT-4-Turbo等）
- 自定义模型输入选项

**服务器配置区块**
- 端口号输入（默认3000）
- 主机地址输入（默认0.0.0.0）
- 日志级别选择

**配置操作**
- 保存配置到.env文件
- 重置为默认配置
- 导出配置备份
- 导入配置文件

**交互细节**
- 输入框 focus 时边框发光效果
- 保存按钮点击后显示loading状态
- 成功保存后显示toast提示
- 配置更改后未保存会有提示

### 4.3 API 测试工具
**请求构建**
- 题目输入（文本框）
- 选项输入（支持多行）
- 题目类型选择（单选/多选/判断/填空）
- 请求方法切换（GET/POST）

**发送请求**
- "发送测试" 按钮
- 显示当前请求的curl命令
- 请求发送中显示loading动画

**响应展示**
- 格式化JSON展示
- 状态码颜色指示（200绿，400橙，500红）
- 响应时间显示
- 答案高亮显示
- 复制响应按钮

**错误处理**
- 网络错误友好提示
- 超时提示（30秒超时）
- API错误信息展示

### 4.4 OCS配置生成器
**功能**
- 自动根据当前配置生成OCS题库配置代码
- 实时预览配置内容
- 语法高亮显示
- 一键复制到剪贴板
- 生成多种格式配置（基础版/完整版）

  method: "get",
  contentType: "json",
  data: {
    title: "${title}",
    options: "${options}",
    type: "${type}"
  },
  handler: `return (res) => { ... }`
}
```

### 4.5 帮助文档
- 快速开始指南
- API接口说明
- 常见问题FAQ
- OCS配置教程
- 可折叠/展开

## 6. Technical Approach

### 前端技术栈
- **纯HTML/CSS/JavaScript** - 无框架依赖，便于集成
- **CSS变量** - 主题管理
- **Fetch API** - HTTP请求
- **LocalStorage** - 配置本地缓存
- **Clipboard API** - 复制功能

### 后端API接口
```
GET  /api/health          - 健康检查
GET  /api/config          - 获取当前配置
POST /api/answer          - 搜题接口（POST）
GET  /api/search           - 搜题接口（GET）
POST /api/config/save     - 保存配置
```

### 数据流
1. 页面加载 → 获取当前配置 → 填充表单
2. 用户修改 → 本地状态更新 → 保存触发API调用
3. 测试请求 → 发送API → 展示响应
4. 生成配置 → 读取表单数据 → 生成代码

### 本地存储
- 最后保存的配置快照
- 请求历史记录
- 用户偏好设置

### 错误处理
- API调用失败：显示友好错误 + 重试选项
- 网络断开：实时检测 + 离线提示
- 配置验证：前端初步验证 + 后端确认

## 7. 关键用户流程

### 流程1: 首次配置
1. 访问配置页面
2. 看到健康检查状态
3. 输入API Key和URL
4. 选择/输入模型
5. 保存配置
6. 测试搜题功能
7. 生成OCS配置代码
8. 复制配置到OCS脚本

### 流程2: 测试API
1. 在API测试工具中输入题目
2. 选择题目类型
3. 点击"发送测试"
4. 查看响应结果
5. 如有问题，修改配置重试

### 流程3: 日常使用
1. 打开配置页面
2. 检查服务状态
3. 如需修改，调整配置
4. 保存并重启服务
