OCS 题库配置代码

将以下配置复制到 OCS 刷题脚本的题库配置中，即可使用 AI 智能搜题功能。 

{
  url: "http://localhost:3000/api/search",
  name: "AI智能搜题",
  homepage: "http://localhost:3000",
  method: "get",
  contentType: "json",
  data: {
    title: "${title}",
    options: "${options}",
    type: "${type}"
  },
  handler: \`return (res) => {
    if (res.code === 1 && res.results && res.results.length > 0) {
      return [res.results[0].question, res.results[0].answer];
    }
    return undefined;
  }\`
}

❓ 使用帮助

🚀 快速开始 ▼

步骤 1: 配置 API

在左侧"API 配置"区域填入你的 OpenAI API Key 和 Base URL。

步骤 2: 测试功能

在"API 测试工具"中输入一道题目，点击"发送测试请求"验证配置是否正确。

步骤 3: 复制配置

将生成的"OCS 题库配置代码"复制到 OCS 刷题脚本的题库配置中。

步骤 4: 开始使用

在 OCS 脚本中启用题库配置，开始自动答题！

🔧 API 配置说明 ▼

API Key

从 OpenAI 平台获取的 API 密钥，格式类似 sk-xxxxxxxxxxxxxxxx

API Base URL

OpenAI API 地址，默认是 https://api.openai.com/v1

如果使用代理或第三方 API，填入对应的地址即可。

支持的模型

  • GPT-3.5 Turbo - 快速、便宜、效果好
  • GPT-4 - 更强能力、更高成本
  • GPT-4 Turbo - 平衡性能和成本
📝 OCS 配置说明 ▼

配置字段说明

  • url - 搜题接口地址
  • name - 题库名称（显示在脚本中）
  • homepage - 题库首页
  • method - 请求方法（get/post）
  • contentType - 响应数据格式
  • data - 发送到接口的参数
  • handler - 处理响应数据的函数

占位符说明

  • ${'{title}'} - 题目标题
  • ${'{options}'} - 题目选项
  • ${'{type}'} - 题目类型
❌ 常见问题 ▼

Q: AI 功能不可用？

A: 检查 API Key 是否正确配置，确保网络可以访问 OpenAI。

Q: 请求超时？

A: AI 生成答案需要几秒钟，请耐心等待。也可以检查网络连接。

Q: 如何部署后端？

A: 可以使用 Docker、PM2 或直接部署。详细说明请查看项目 README。

Q: 支持其他 AI 吗？

A: 支持任何兼容 OpenAI API 格式的服务，如 Azure OpenAI、LocalAI 等。
