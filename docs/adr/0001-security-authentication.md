# ADR-0001: 安全认证与 API Key 管理系统

## 状态

已通过

## 背景

OCS AI 搜题后端服务需要增加安全机制，包括管理员账户认证和 API Key 管理，防止未授权访问并实现调用统计。

## 决策

### 1. 用户架构
- **管理员**：拥有 Web 管理界面访问权限，可管理 AI 配置和 API Keys
- **OCS 客户端**：仅通过 API Key 访问搜题接口，无需账户

### 2. 管理员认证
- 首次启动时自动创建默认管理员账户
- 默认用户名：`admin`，默认密码：`admin123`
- 首次登录强制要求修改密码（`mustChangePassword` 标记）
- 密码采用明文存储（简化实现）
- 会话使用简化 Token 机制，存储于 localStorage

### 3. API Key 管理
- **格式**：采用别名 + 完整 Key 双字段
  - `alias`：简短易识别的名称（如 `ocs_key_1`）
  - `key`：完整 Key 值（如 `sk_xxxxxxxxxxxx`）
- **限制**：支持过期时间（`expiresAt`）和使用次数（`maxCalls`）限制
- **操作**：支持创建、查看、编辑、删除、禁用/启用、重新生成
- **传递**：通过请求体字段 `apiKey` 传递

### 4. API Key 校验规则
校验顺序：
1. 检查请求是否包含 `apiKey` 字段（**必须**）
2. 检查 API Key 是否存在且未被禁用
3. 检查是否过期（无过期时间则跳过）
4. 检查剩余次数是否充足（无次数限制则跳过）

### 5. 调用统计
- 仅记录每个 API Key 的**总调用次数**
- 不记录调用明细（简化实现）

### 6. 向后兼容
- 完全移除旧 `/api/answer` 接口
- 所有请求必须携带有效 API Key
- 需要更新所有 OCS 客户端配置

### 7. 错误响应格式
统一使用错误码：
```json
{
  "code": 401,
  "msg": "API Key 无效或已过期"
}
```

## 后果

### 正面影响
- 防止未授权访问服务
- 支持多 OCS 客户端独立管理
- 可追踪 API 使用情况
- 可控制 API 访问权限（禁用/过期）

### 负面影响
- 所有 OCS 客户端需要更新配置（添加 API Key）
- 增加了部署和配置复杂度
- 明文密码存储安全性较低（但适用于内部服务场景）
